1. Root Cause Analysis
--------------------------------------------------------------------------------
The previous implementation invoked separate prompt functions recursively or 
sequentially, creating nested readline instances and duplicated keypress event 
listeners. When returning "Back" to a previous step, temporary listeners from 
prior executions were not fully closed or were double-registered. Consequently,
on subsequent Back actions, keypress handlers conflicted and events were dropped.

--------------------------------------------------------------------------------
2. Lifecycle & State Machine Strategy
--------------------------------------------------------------------------------
- Unified Navigation Controller:
  Implemented `runInitWizardStateController` to act as a single centralized 
  wizard state machine.
- Single Event Listener Lifecycle:
  One single `keypress` event listener is registered on `process.stdin` for the 
  entire duration of the interactive setup wizard step.
- State-Based Transitions:
  Rather than calling recursively, navigation changes state (`currentStep = 'setup'`
  or `currentStep = 'details'`) and refreshes/erases panels cleanly.
- Confirmed Clean Event Disposals:
  When the wizard is submitted (Enter) or aborted (Ctrl+C), it fully removes the 
  listener and restores the raw-mode terminal configuration. Temporary 
  disablings are performed during normalization prompts to prevent conflicts.
